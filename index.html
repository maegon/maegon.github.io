ABOUT ME
임재원 JAVA BE 개발자
소개말 안녕하세요. JAVA 백엔드 신입 개발자 임재원입니다.
저는 Spring Framework와 Java를 기반으로 한 프로젝트를 통해 탄탄한 기초와 실무 경험을 쌓아왔습니다.

새로운 기술을 배우고 기능을 구성할 때
해당 기술을 사용하면 어떨까?,

내가 상상한 그림을 실현하는 코딩에
재미를 느끼는 개발자입니다.
목표 사용자에게 다양한 경험을 제공하고
팀워크와 협업을 중시하는 성장형 개발자,

제가 상상한 결과물을 실현하기 위해
새로운 기술을 받아들이고 이를 적용하려
노력하는 개발자가 되고자 합니다.
최종학력
세한대학교 항공교통물류학과 (졸업)

자격증
정보처리기능사, 운전면허 1종 보통, 컴퓨터활용능력 2급
SKILLS
스프링 기반 웹솔루션 풀스택 과정에서 백엔드와 프론트엔드 개발을 다루며 다양한 기술 스택을 학습했습니다. 인텔리제이 환경에서 자바, 스프링 부트 프레임워크, Node.js의 백엔드 기술을 사용하고 HTML5, CSS, JavaScript, 부트스트랩, Thymeleaf, Tailwind, jQuery, Ajax를 사용하여 프론트엔드 기술을 다뤄봤습니다. MySQL, mariaDB를 통해 데이터베이스를 관리하였습니다. 학습한 각 기술에 대한 숙련도를 5점에 가까울 수록 고급 기술을 사용할 줄 알고 추가적인 자료를 참고하면 부족한 부분도 스스로 해결할 수 있는 능력을 가진 것을 기준으로 표시하였습니다.
HTML5
Thymeleaf를 사용하여 동적인 웹 페이지를 구성하고 시맨틱 태그를 활용할 수 있음. 스프링 부트와 Thymeleaf를 함께 사용하여 웹 개발할 수 있음.
CSS
Tailwind와 같은 유틸리티 클래스 기반의 프레임워크를 선호하며 효율적인 CSS 작성 가능. 다만, 더 복잡한 CSS 애니메이션이나 고급 CSS 트릭은 잘 다루지 못함.
JavaScript
기본적인 로직 구현과 프론트엔드-백엔드 간 데이터 통신을 처리할 수 있지만, React, Sveltekit와 같은 프레임워크를 사용한 고급 JavaScript 패턴을 잘 다루지 못함.
반응형 웹 디자인
Tailwind CSS를 사용하여 반응형 디자인을 구현할 수 있지만 디바이스에서 레이아웃이 망가지지 않도록 조정하는데 능숙치 못함.
HTML, CSS, JavaScript를 사용하여 웹 페이지를 만들고, 반응형 웹 디자인을 구현할 수 있습니다. 그러나 React, Sveltekit 등과 같은 프론트엔드 프레임워크에 대한 경험이 많지 않고, CSS의 고급 애니메이션이나 JavaScript의 복잡한 로직 구현에 대한 부분은 추가 학습이 필요합니다.
Java/Spring Boot
자바의 핵심 문법과 객체지향 프로그래밍에 능숙하며, 스프링 부트와 같은 프레임워크를 활용하여 실무 수준의 웹 애플리케이션을 개발할 수 있음. 자바 9 이상의 고급 기능이나 최신 자바 기능에 대한 경험은 조금 더 학습이 필요함. 스프링 부트의 의존성 주입(DI), MVC 패턴 활용, 데이터베이스 연동 및 Thymeleaf을 통한 뷰 레이어 통합할 수 있음.
데이터베이스
스프링 부트와 함께 JPA/Hibernate를 사용하여 데이터베이스와의 연동을 할 수 있음. 엔티티와 테이블 매핑, CRUD 연산, 간단한 SQL 쿼리를 작성할 수 있지만 복잡한 쿼리나 성능 최적화, 데이터베이스 트랜잭션 관리에 대해 추가적인 학습이 필요함.
API 개발
RESTful API를 설계하고 JSON을 사용하여 클라이언트와 서버 간 데이터를 교환하고 Ajax를 통해 비동기 통신을 처리할 수 있지만 GraphQL과 같은 API 설계 방식이나 고급 RESETful API 설계 패턴에 대한 경험이 없음.
배포 및 운영
네이버 클라우드 플랫폼, Termius, CI/CD 파이프라인, Docker 컨테이너, Jenkins, GitHub Actions를 활용한 스프링 부트 애플리케이션의 기본적인 배포 방법을 직접 경험해보았지만 배포에 대한 추가적인 학습이 필요함.
스프링 부트를 사용하여 웹 애플리케이션을 개발하고 데이터 베이스와 연동하며, RESTful API를 구축하는데 익숙합니다. 그러나 보안 및 고급 아키텍처 설계, 클라우드 배포 등에서 더 고급 기술을 적용하는 부분은 추가 학습이 필요합니다.
Git
깃의 기본적인 커맨드(git init, git add, git commit, git push, git pull, git clone 등)을 사용하여 로컬 저장소와 원격 저장소를 관리할 수 있음. 프로젝트의 변경 사항을 추적하고, 브랜치(git branch, git checkout, git merge)를 활용하여 여러 작업을 분리하여 관리하는 것에 익숙함.
branch 및 협업
branch를 사용하여 작업을 분리하고 branch 병합하여 변경 사항을 통합하는 작업을 수행할 수 있고 협업을 위한 깃허브 워크플로우, 예를 들어, 풀 리퀘스트(Pull Request)를 생성하고 검토하는 것, 코드 리뷰를 통한 협업을 경험해봤으며 충돌 에러에 대한 대처를 할 수 있음.
고급 Git 및 깃허브 기능
깃의 고급 기능으로 git rebase 사용은 익숙하나, git cherry-pick, git stash, git bisect에 대한 사용을 적극적으로 하지 않아 익숙치 않음. 깃허브의 고급 기능인 GitHub Actions를 배포 구현 시 사용한 적이 있으나 추가 학습이 필요함.
문서화 및 프로젝트 관리
깃허브 리포지터리의 README 파일에 마크다운을 사용하여 프로젝트를 소개할 수 있으나 깃허브를 통한 포트폴리오 프로젝트의 체계적인 문서화, 예를 들어 깃허브 위키(GitHub Wiki)나 깃허브 프로젝트(GitHub Projects) 기능을 활용한 고급 문서화 및 프로젝트 관리 경험이 없음.
깃과 깃허브의 기본적인 사용법을 알고, 로컬 및 원격 저장소 관리, 브랜치 활용, 간단한 협업을 수행할 수 있습니다. 깃 고급 기능 및 프로젝트 문서화를 위한 깃허브의 고급 기능에 대해 추가 학습이 필요합니다.
